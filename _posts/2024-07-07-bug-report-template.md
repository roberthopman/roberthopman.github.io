A good bug report can save hours of debugging.

[The gist at github](https://gist.github.com/roberthopman/96154ffab30e89d2972cb875d09ecec8)

```md
---
name: Bug report
about: Create a report to help us improve
title: ''
labels: ''
assignees: ''

---

**Describe the bug**
A clear and concise description of what the bug is.

**Environment**
- Device: [Desktop, Smartphone, Tablet]
- OS: [e.g., Windows 10, macOS 11.2]
- Browser: [e.g., Chrome 91, Firefox 89]
- Application Version: [e.g., 1.2.3]
- Configuration: [e.g., Tenant: ConstructionGroup, User: Email, Role: Admin, Feature flag: Enabled]

**Expected behavior**
- [Describe what you expected to happen]

**Actual behavior**
- [Describe what actually happened]

**To reproduce**

0. Describe the initial state of the application
1. Go to '...'
2. Click on '....'
3. Scroll down to '....'
4. See error

**Reproducibility**
- [e.g., Happens every time, Intermittent (1 out of 5 times)]

**Additional information**
- [Include screenshots, videos, logs, error messages, etc.]
- [Mention any recent changes or related issues]
```
